## 一、 关于产品设计

### 1. 五子棋的规则细化
在标准五子棋（Gomoku）中，有两种主流规则体系：
* **无禁手规则（自由规则）**：
  * **规则描述**：黑白双方轮流落子，任意一方最先在横、竖、斜任意方向连成 **5 个或 5 个以上** 同色棋子即获胜。
  * **特点**：简单直观，但黑棋（先手）具有极大的必胜优势。
  * **建议**：在目前 **v1/v2 阶段优先使用无禁手规则**，确保游戏主流程极简、流畅。
* **禁手规则（专业/现代规则，如 Renju 禁手）**：
  * **规则描述**：为了限制黑棋先手优势，黑棋禁止下出“三三”、“四四”、“长连”（连子超过5个）。若黑棋下出禁手，则直接判负。白棋无任何禁手限制，且长连算赢。
  * **后续考量**：在 v3/v4 版本中可作为“专业模式”开关提供。

### 2. 棋局保存设计（为 v4 拟人化 AI 做准备）
v4 版本需要通过记录不同人的对局数据来模拟不同人的风格。因此，棋局数据的持久化设计至关重要。

推荐采用以下两种保存格式：
* **方案 A：JSON 格式棋谱 (推荐用于 Web/LLM API 交互)**
  * **格式示例**：
    ```json
    {
      "game_id": "20260615_235800_abc",
      "date": "2026-06-15 23:58:00",
      "players": {
        "black": "Human_Player",
        "white": "AI_Persona_A"
      },
      "board_size": 15,
      "moves": [
        {"step": 1, "player": 1, "pos": [7, 7], "time_spent": 2.5},
        {"step": 2, "player": 2, "pos": [7, 8], "time_spent": 1.2}
      ],
      "result": {
        "winner": 1,
        "reason": "five_in_a_row"
      }
    }
    ```
  * **优点**：极易与 Python/Web 前端（JS）交互，结构清晰，方便提取对棋手下子习惯的分析（例如落子时间、开局偏好、防守/进攻倾向等），为 v4 拟人化 AI 训练或 Prompt 分析提供最直接的特征。
* **方案 B：SGF 标准棋谱格式 (Smart Game Format)**
  * 围棋、五子棋领域的国际通用标准格式。
  * **优点**：能够无缝兼容现有的五子棋复盘软件和专业打谱工具。

## 二、 关于测试：测试该怎么做？

为了确保在敏捷迭代（v1 -> v4）过程中，修改代码不会引入新的 Bug（尤其是胜负判断和越界逻辑），我们应引入以下测试维度：

### 1. 单元测试 (Unit Test) —— 针对核心逻辑
* **测试对象**：`board.py` 中的核心算法。
* **具体测试用例**：
  * **落子有效性测试**：边界测试（在 `(0,0)`, `(14,14)` 落子是否成功；在 `(-1,0)`, `(15,15)` 落子是否抛出异常或拦截）。
  * **覆盖判定测试**：在已有棋子的格子上落子，是否返回 `False`。
  * **胜负判定测试 (核心)**：
    * 横向五连子：模拟在 `(7,5)` 至 `(7,9)` 连续落子，验证 `check_winner` 是否返回 `True`。
    * 纵向五连子：验证纵向五连子。
    * 主对角线、副对角线五连子。
    * 未达五连子时（如4个子）验证是否返回 `False`。
* **测试框架**：使用 Python 内置的 `unittest` 库，编写 `test_board.py`。

### 2. 模拟对局测试 (Simulation Test)
* 编写测试脚本，模拟一组预先定义好的棋步序列（如著名的“四三连”攻击序列），自动运行到终局，并断言最终胜者是否符合预期。
